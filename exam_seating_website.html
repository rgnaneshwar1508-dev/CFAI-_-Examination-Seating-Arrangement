Project abstract

What is ExamSeat?

The Examination Seating Arrangement system allocates students to exam seats while avoiding adjacency conflicts and meeting special requirements. The primary objective is to ensure fair and secure seating by preventing students from the same class or subject from sitting next to each other.


The system also handles special needs such as accessible seating for differently-abled students or reserved seats for specific conditions. When constraints increase, the arrangement can become over-constrained — no valid seating pattern satisfies all requirements simultaneously.


The system identifies such situations and explains the reason for failure: insufficient seats, too many restrictions, or conflicting conditions. The solution uses CSP and graph-based algorithms to generate efficient, explainable seating plans, improving exam management and reducing manual errors.

Algorithm

How the solver works

01
Problem formulation (CO1)
The hall is a grid graph. Each seat is a variable, each student a domain value. Adjacency rules become hard constraints. Accessible seats are special-case constraints handled before the main solver runs.
02
Variable ordering — MRV (CO3)
Seats with the fewest valid student candidates are selected first. Ties are broken by the degree heuristic — seats with more constrained neighbours go first.
03
Value ordering — LCV (CO3)
For each seat, students are tried in Least Constraining Value order — placing the student that rules out the fewest options for neighbouring seats first.
04
Forward checking & backtracking (CO2, CO3)
After each assignment, forward checking prunes inconsistent values from neighbouring domains. If a domain empties, the solver backtracks and logs the failure reason.
05
Over-constraint detection (CO3, CO6)
If no valid assignment exists, the system explains why: too many students from the same class, insufficient seats, or conflicting adjacency requirements.
06
Risk scoring & report (CO5, CO6)
A Bayesian network scores each seat pair for cheating risk. The final report includes the seat map, constraint proof, inference steps, and flagged high-risk adjacencies.
CSP solver — Python pseudocode
def solve_seating(hall, students, constraints):
  # CO1: Formulate as CSP
  csp = CSP(
    variables=hall.seats,
    domains={s: students[:] for s in hall.seats},
    constraints=constraints)

  # CO3: MRV + Degree heuristic
  def select_seat(unassigned, domains):
    return min(unassigned, key=lambda s:
      (len(domains[s]), -degree(s)))

  # CO3: LCV ordering
  def order_values(seat, domain):
    return sorted(domain,
      key=lambda st: conflicts_caused(seat, st))

  # CO2: Backtracking + forward check
  def backtrack(assignment):
    if complete(assignment): return assignment
    seat = select_seat(unassigned, domains)
    for student in order_values(seat, domains[seat]):
      if consistent(seat, student, assignment):
        assignment[seat] = student
        inf = forward_check(seat, student)
        if inf != FAILURE:
          result = backtrack(assignment)
          if result: return result
        restore_domains(inf)
    return FAILURE

  # CO5: Bayesian risk scoring
  sol = backtrack({})
  if sol: return score_risk(sol, bayes_net)
  else: raise OverConstrainedError(explain())
System capabilities

What the system can do

🗺️
Conflict-free seat maps
Automatically generates seating plans where no two students from the same class or subject are adjacent — horizontally, vertically, or diagonally.
♿
Accessibility support
Reserves front-row or aisle seats for differently-abled students before running the main solver. Treated as hard constraints with highest priority.
⚡
Over-constraint detection
When no valid arrangement exists, the system identifies the exact reason — same-class overload, insufficient seats, or conflicting rules — and explains it clearly.
🔍
Step-by-step reasoning trace
Every assignment, constraint check, forward-check result, and backtrack is logged with a full trace — satisfying CO6 explainability requirements.
📊
Heuristic comparison
Run MRV, Degree, or LCV variants independently and compare backtrack counts, time taken, and nodes expanded — satisfying CO2's empirical analysis.
🧠
Bayesian risk scoring
After placement, a Bayesian network estimates cheating risk for each adjacent pair based on class proximity, subject overlap, and exam history (CO5).
